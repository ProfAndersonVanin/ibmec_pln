role para explorar
Conceito fundamental

O que é o hidden state?

O hidden state (hₜ) é um vetor de números que a RNN carrega de uma palavra para a próxima. Funciona como um bloco de notas que a rede atualiza a cada passo, misturando a palavra atual com tudo que foi lido antes.

ht = tanh( W × xt + U × ht-1 + b )
hₜ = novo hidden state (memória atualizada) xₜ = palavra atual (embedding) W = pesos da palavra U = pesos da memória tanh = mantém valores entre −1 e +1
Demonstração interativa

Hidden state atual — h₀ (início)
Estado inicial
O hidden state começa zerado. A RNN não tem nenhuma memória ainda — como uma folha em branco antes da leitura.
Memória acumulada
Como acontece

Os quatro passos a cada palavra

Passo 01
Embedding
A palavra é convertida em um vetor numérico — o embedding. "gato" vira [0.7, -0.2, 0.5, ...]
xₜ = W_embed[id_palavra]
Passo 02
Contribuição da palavra
O embedding é multiplicado pela matriz de pesos W, gerando a influência da palavra atual.
contrib_x = W × xₜ
Passo 03
Contribuição da memória
O hidden state anterior é multiplicado pela matriz U — a "memória" influencia o novo estado.
contrib_h = U × hₜ₋₁
Passo 04
Combinação + tanh
As duas contribuições são somadas e passadas pela função tanh, que comprime tudo entre −1 e +1.
hₜ = tanh(contrib_x + contrib_h + b)
Limitação da RNN

O problema do esquecimento

Quanto mais longa a frase, mais a RNN "esquece" o que aconteceu no início. A influência das primeiras palavras se dilui a cada novo passo — isso é o vanishing gradient.

Influência de cada palavra no hidden state final
Tamanho do círculo = influência restante no estado final. Vermelho = quase esquecida. Verde = ainda influente.
RNN simples
Uma memória só. Cada novo passo sobrescreve parte da memória anterior. Em frases longas, o início é progressivamente perdido.
LSTM (solução)
Duas memórias: cell state (longo prazo) + hidden state (curto prazo). Três portões controlam o que manter, o que esquecer e o que usar.
Resumo

O que aprendemos

Conceito
hₜ = memória viva
O hidden state não é fixo — ele muda a cada palavra, acumulando contexto ao longo da sequência.
Mecanismo
Mistura dupla
Cada novo estado mistura a palavra atual (xₜ) com a memória anterior (hₜ₋₁) através de matrizes de pesos aprendidas.
Limitação
Esquecimento gradual
A influência de palavras distantes decai. Frases longas são problemáticas — motivando a criação da LSTM em 1997.
Prático
return_sequences
No Keras, return_sequences=True retorna todos os hidden states — não só o final — para inspecionar cada passo.
